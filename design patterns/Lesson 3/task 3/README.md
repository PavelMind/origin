#### Паттерн "цепочка ответственности".
Есть класс сообщения о логировании с типом сообщения и самим сообщением. Всего есть четыре типа сообщений: предупреждение, ошибка, фатальная ошибка и неизвестное сообщение.  
Есть обработчики для разных типов, которые будут обрабатывать сообщения по приоритету:
1. фатальная ошибка,
2. ошибка,
3. предупреждение,
4. неизвестное сообщение.  

Каждый обработчик обрабатывает сообщение только определённого типа, и передаёт сообщение следующему обработчику, если оно было не для него.  
Что обработчик умеет в зависимости от типа сообщения:
- Обработчик фатальной выбросить исключение с текстом сообщения, а программа должна прекратить выполнение после возникновения фатальной ошибки.
- Обработчик обычной ошибки должен записать её в файл по указанному пути.
- Обработчик предупреждения должен напечатать сообщение в консоль.
- Обработчик неизвестного сообщения должен выбросить исключение с текстом о необработанном сообщении.
